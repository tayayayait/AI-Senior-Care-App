

# 5060 AI 시니어 케어 앱 구현 계획

## 개요
상세서.md의 디자인 토큰과 UI/UX 규격을 기반으로, **모바일 사용자 화면(웹뷰 반응형)** + **관리자 웹 대시보드**를 모두 구현합니다. AI 대화는 **UI만 더미 데이터**로 먼저 구현하고, 실제 연동은 이후 단계에서 진행합니다.

---

## 1단계: 디자인 시스템 세팅

### 디자인 토큰 적용
- **컬러 시스템**: Primary(#0F766E 계열 teal), Secondary(#D97706 amber), Neutral(slate 계열), Emergency(#B91C1C), Semantic(info/success/warning/danger) 등 전체 팔레트를 CSS 변수로 등록
- **타이포그래피**: Pretendard 폰트 적용, type.display(28px) ~ type.caption(12px) 스케일 정의, 본문 17px/줄간 1.55 기본
- **간격/크기**: 4px 그리드 기반 spacing, 터치 영역 최소 44×44px, 주요 CTA 48px+
- **라운드/그림자**: radius.sm(8px), radius.md(12px), radius.lg(16px), shadow.1~3
- **모션**: 120ms(fast) ~ 300ms(page), prefers-reduced-motion 대응

### 컴포넌트 커스터마이징
- Button: L(52px), M(48px), S(40px) 높이, Primary/Secondary/Ghost/Destructive 변형, 12px 라운드
- Input: 52px 높이, 17px 텍스트, 라벨 필수 표시, 포커스 링 #2DD4BF
- Textarea: 최소 120px ~ 최대 240px, Input과 동일 스타일
- Card: 16px 패딩/라운드, shadow.1, 흰색 배경
- Modal: 스크림 48%, 16px 라운드, 포커스 트랩
- Toast: 상단 12px, 3.5초 지속, 최대 3개 스택

## 2단계: 모바일 사용자 화면 (반응형 390px 기준)

### 2-1. 온보딩/스플래시
- 환영 메시지가 있는 따뜻한 톤의 스플래시 화면
- 앱 소개 슬라이드 (2~3장, AI 감성 대화·긴급 안전망 소개)
- "시작하기" CTA 버튼 (full-width, 52px 높이)

### 2-2. 로그인/회원가입
- 전화번호 기반 간편 로그인 화면 (5060 맞춤 큰 입력 필드)
- 회원가입: 이름, 전화번호, 비상 연락처 입력
- 폼 라벨 항상 표시, placeholder로 라벨 대체 금지

### 2-3. 홈 화면
- **상단 앱바** (56px): 프로필 아이콘 + 앱 로고
- **인사 카드**: "OO님, 오늘 하루는 어떠셨나요?" 따뜻한 감성 인사
- **빠른 메뉴 카드들**: AI 대화 시작, 오늘의 건강 팁, 긴급 연락
- **긴급 신고 버튼**: 하단 고정, 64px 높이, #B91C1C 배경, 항상 접근 가능
- **하단 탭바** (72px): 홈, AI대화, 안전, 설정 (아이콘 24px + 라벨 12px)

### 2-4. AI 감성 대화 (Chat)
- **채팅 인터페이스**: 
  - 사용자 말풍선: bg #0F766E, text 흰색, 최대 폭 78%
  - AI 말풍선: bg 흰색, border #E2E8F0, 최대 폭 78%
  - 말풍선 라운드 16px, 패딩 12px, 꼬리 없는 단순 형태
  - 시간 표시: 12px, #475569, 메시지 아래 4px 간격
- **입력 영역**: min 64px, 키보드 위 고정
- **타이핑 인디케이터**: 점 3개 애니메이션 (6px dot, 180ms)
- **위험 키워드 감지 팝업**: 
  - 경고 배너 #D97706 (amber)
  - 모달: "도움 받기" (Primary) + "괜찮아요" (Secondary)
  - 바깥 클릭 닫힘 금지 (실수 방지)
- 더미 대화 데이터로 UI 시연

### 2-5. 긴급 신고 화면
- **119 연결 버튼**: 64px 높이, #B91C1C, "119 연결" 명확한 라벨 (18px bold)
- **경찰 연결 버튼**: 별도 CTA
- **2단계 안전장치**: 
  1. 확인 모달 ("정말 신고하시겠습니까?")
  2. 3초 카운트다운 (취소 가능)
- 비상 연락처 목록 표시
- 실패 시 직접 전화번호 표시 + 오류 토스트 5초

### 2-6. 설정 화면
- 프로필 정보 수정
- 비상 연락처 관리
- 글자 크기 조절
- 알림 설정
- 회원 탈퇴 (2-step 확인)

## 3단계: 관리자 웹 대시보드 (1024px+ 데스크톱 기준)

### 3-1. 레이아웃
- **사이드바**: 280px 고정 (1024px 이상), 1024px 미만은 Drawer 전환
- **본문 컨테이너**: max-width 1200px, 중앙 정렬
- **12컬럼 그리드**, gutter 24px

### 3-2. 대시보드 메인
- 주요 지표 카드: 총 사용자 수, 오늘 대화 수, 위험 감지 건수, 긴급 신고 건수
- 최근 위험 감지 알림 목록
- 최근 가입 사용자 목록

### 3-3. 사용자 관리
- 데이터 테이블: 헤더 44px sticky, 행 44px, zebra 배경, 페이징 20건
- 검색/필터 기능
- 사용자 상세 정보 모달
- 빈 상태: "데이터 없음" + 필터 초기화 CTA

### 3-4. 대화 모니터링
- AI 대화 로그 목록 (테이블)
- 위험 키워드 감지 내역 강조 표시
- 대화 상세 보기 (채팅 형태로 표시)

### 3-5. 긴급 신고 관리
- 신고 내역 테이블
- 상태별 필터 (접수/처리중/완료)
- 실시간 알림 배지 + toast

## 4단계: 라우팅 & 네비게이션

### 모바일 라우트
- `/` → 온보딩/스플래시
- `/login` → 로그인
- `/signup` → 회원가입
- `/home` → 홈 화면
- `/chat` → AI 감성 대화
- `/emergency` → 긴급 신고
- `/settings` → 설정

### 관리자 라우트
- `/admin` → 대시보드
- `/admin/users` → 사용자 관리
- `/admin/conversations` → 대화 모니터링
- `/admin/emergencies` → 긴급 신고 관리

---

## 기술 사항
- 모바일 뷰는 max-width 414px 반응형으로 구현 (실제 앱처럼 보이도록)
- 모든 데이터는 더미 데이터 (JSON mock)로 구현
- Pretendard 웹폰트 로드
- prefers-reduced-motion 미디어 쿼리 대응
- 상세서의 디자인 토큰을 CSS 변수로 일관되게 적용
