# AI Senior Care App UI/UX 상세서 (명세 v1.0)

본 문서는 **5060세대 맞춤형 감성 대화 + 생활 안전망** 모바일 앱(Android/iOS) 및 관리자(Web) 화면을 **동일한 규칙/토큰**으로 구현하기 위한 UI/UX 구현 기준입니다.  
용어: **필수(MUST)** / **권장(SHOULD)** / **허용(MAY)**.

## 1) 디자인 원칙(5060 맞춤)

| 항목 | 값(HEX·px·ms) | 사용처 | 규칙 |
|---|---:|---|---|
| 가독성 최우선 | 본문 17px, 줄간 1.55 | 전 화면 | 본문 기본 크기 17px 유지, 16px 미만 금지(예외: 캡션 12px) |

## 4) 컴포넌트 공통 상태(State) 규칙

> 아래 상태는 모든 컴포넌트에 적용되는 “기본 규칙”입니다. 각 컴포넌트 섹션에서 색/레이아웃만 덮어씁니다.

| 항목 | 값(HEX·px·ms) | 사용처 | 규칙 |

| `state.active` | `overlay 10%` | 공통 | 누름/프레스 시 10% 오버레이(모바일은 active만 사용) |

| 실패 처리 | `error toast 5000ms` | 긴급 | 통화 불가/권한 거부 시 대체 안내(직접 전화번호 표시) |

## 7) 접근성(A11y) 체크리스트(필수)

| 항목 | 값(HEX·px·ms) | 사용처 | 규칙 |
|---|---:|---|---|
| 텍스트 대비 | `>= 4.5:1` | 공통 | 본문/라벨은 4.5:1, 큰 텍스트(>=18px bold or 24px) 3:1 |
| 터치 타깃 | `>= 44×44px` | 공통 | 아이콘 버튼도 포함 |
| 포커스 표시 | `ring 3px #2DD4BF` | 키보드 | 포커스가 “보이는” 상태여야 함 |
| 폼 라벨 | `label 항상 표시` | 폼 | placeholder로 라벨 대체 금지 |
| 오류 안내 | `색+텍스트` | 폼 | 오류를 색만으로 전달 금지, 해결 방법 포함 |
| 스크린리더 | `aria-label` | 아이콘 버튼 | 아이콘만 있는 버튼은 접근성 라벨 필수 |
| 동적 글자 크기 | `100~200%` | 앱 | OS 글자 확대 시 레이아웃 깨짐 없이 리플로우 |
| 모션 감소 | `reduce motion` | 공통 | OS 설정 시 비필수 애니메이션 제거 |
| 알림/경고 | `role=alert` | 위험/오류 | 중요 알림은 스크린리더에 즉시 전달 |
| 시간 제한 | `연장 제공` | 세션/보안 | 자동 로그아웃/제한 있는 경우 연장/저장 제공 |

## 8) UI QA 체크리스트(필수)

| 항목 | 값(HEX·px·ms) | 사용처 | 규칙 |
|---|---:|---|---|
| 대표 해상도 테스트 | `360/390/414/768/1024/1280` | 공통 | 최소 6개 폭에서 레이아웃 확인 |
| 안전영역(safe-area) | `top/bottom inset` | iOS | 노치/홈 인디케이터에 UI 겹침 금지 |
| 키보드 회피 | `input always visible` | 앱 | 채팅/폼에서 입력창 가림 금지 |
| 로딩 상태 | `skeleton/spinner` | 공통 | 모든 API 호출에 loading 정의(빈 화면 금지) |
| 오류 상태 | `error state` | 공통 | 네트워크/권한/서버 오류 각각 다른 안내 |
| 오프라인 | `retry CTA` | 공통 | 오프라인 안내 + 재시도 버튼 |

| 위험/긴급 플로우 | `2-step 확인` | 안전망 | 실수 방지, 취소 가능, 로그 기록 |

## 9) 가정(Assumptions)

| 항목 | 값(HEX·px·ms) | 사용처 | 규칙 |
|---|---:|---|---|
| 기본 모드 | `Light mode` | 전체 | 다크모드는 v2 범위(요청 시 별도 토큰 추가) |
| 기본 폰트 적용 | `Pretendard 사용 가능` | 앱/관리자 | 번들/서버 로드 가능하다고 가정(불가 시 시스템 폰트) |
| 디자인 단위 | `px 기준` | 공통 | iOS/Android는 dp/pt로 변환 적용 |
| 관리자 UI | `Web(Desktop 우선)` | 관리자 | 1024px 이상을 기본 기준으로 설계 |
| 테이블/차트 | `표 중심` | 관리자 | 차트는 v2에서 추가(현재 표/지표 카드 중심) |
